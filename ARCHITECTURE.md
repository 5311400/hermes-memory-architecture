# Architecture

## 核心理念

这是一个**分层记忆系统**，灵感来自人类记忆的层次结构：

- **工作记忆（L1-L2）**— 当前对话中即时需要的信息
- **情景记忆（L3-L5）**— 最近发生的事、可重用的过程、关联关系
- **长期记忆（L6-L8）**— 归档数据、行为模式、深度知识

每层独立工作，但通过数据流串联成完整链路。

## 访问模式

**日常写入路径：**
```
用户对话 → L1 (memory add) → L3 (diary.py) → L5 (entity_links)
                                                → L7 (MemOS 自动提取)
                                                → L8 (GBrain 事实提取)
```

**日常读取优先级：**
```
L1 (开机记忆) → L2 (最近会话) → L3 (今天/昨天日记)
    → L4 (相关技能) → L7 (行为轨迹) → L5 (关联实体) → L8 (知识图谱)
```

## 设计决策

### 为什么不用单一方案？
- 单一方案（比如只用向量数据库）在手机上跑不动（ARM64 无 sqlite-vec）
- 各层关注点不同：L1 追求速度，L3 追求可靠同步，L7 追求行为理解，L8 追求深度推理
- 每层独立替换：Hindsight → MemOS 迁移时不影响其他层

### 为什么冷热分离？
- 手机存储有限（256GB），3TB 路由器硬盘适合冷数据
- 行为轨迹（L7）不需要直接搜索，只在需要时调取
- 知识图谱（L8）在电脑端运行，不受手机资源限制

### 为什么不用云端存储？
- 用户隐私偏好，所有数据本地优先
- 只在必要的同步场景使用 IMA（日记笔记）

## 演进历史

```
初始状态 → Hindsight (单层记忆，5000字节限制)
    ↓
L1-L5 初步建成 → 分层意识形成
    ↓
R3D WebDAV 接入 → 冷存储层 (L6)
    ↓
Hindsight → MemOS v2.0 迁移 → 行为+世界模型 (L7)
    ↓
WSL 电脑端上线 → GBrain 知识图谱 (L8)
    ↓
当前：8层全链路运行
```
